## 后端技术栈

### 核心技术
- **Express.js**: 轻量级 Node.js Web 框架
- **SQLite**: 嵌入式关系型数据库
- **better-sqlite3**: 同步 SQLite3 驱动，性能更好
- **CORS**: 跨域资源共享中间件

### 为什么选择这个技术栈？

| 技术 | 优点 | 适用场景 |
|------|------|----------|
| Express | 轻量、灵活、易扩展 | 小型到中型应用 |
| SQLite | 零配置、跨平台、无需安装 | 本地存储、离线应用 |
| better-sqlite3 | 同步API、性能高 | 高并发读写操作 |

### 2. 数据库设计

```sql
-- 桥梁数据表
CREATE TABLE bridges (
    id INTEGER PRIMARY KEY AUTOINCREMENT,  -- 自增主键
    name TEXT NOT NULL,                   -- 桥梁名称
    dynasty TEXT,                          -- 建造朝代
    year INTEGER,                          -- 建造年份
    province TEXT,                         -- 所在省份
    location TEXT,                         -- 详细地址
    type TEXT,                             -- 桥梁类型
    length REAL,                           -- 桥长（米）
    span REAL,                             -- 跨度（米）
    material TEXT,                         -- 建造材料
    city TEXT,                             -- 所在城市
    technology TEXT,                       -- 施工工艺
    description TEXT,                      -- 描述信息
    image TEXT,                            -- 图片路径
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP  -- 创建时间
);

-- 索引优化查询性能
CREATE INDEX idx_dynasty ON bridges(dynasty);
CREATE INDEX idx_province ON bridges(province);
CREATE INDEX idx_material ON bridges(material);
CREATE INDEX idx_technology ON bridges(technology);
```

### 4. 核心功能实现

#### 4.1 条件筛选
```javascript
// 支持多种筛选条件
const { search, dynasty, province, material, technology } = req.query;

let whereClauses = [];
let params = [];

// 模糊搜索
if (search) {
  whereClauses.push('(name LIKE ? OR location LIKE ?)');
  params.push(`%${search}%`, `%${search}%`);
}

// 精确筛选
if (dynasty) {
  whereClauses.push('dynasty = ?');
  params.push(dynasty);
}

// 组合 WHERE 子句
const whereSQL = whereClauses.length > 0 
  ? 'WHERE ' + whereClauses.join(' AND ') 
  : '';
```

#### 4.2 统计分析
```javascript
// 按朝代统计示例
app.get('/api/bridges/by-dynasty', (req, res) => {
  const stmt = db.prepare(`
    SELECT dynasty as name, COUNT(*) as value 
    FROM bridges 
    GROUP BY dynasty 
    ORDER BY value DESC

  const data = stmt.all();
```

#### 4.3 数据事务
```javascript
// 批量插入使用事务保证数据一致性
const insertMany = db.transaction((bridges) => {
  for (const bridge of bridges) {
    stmt.run(
      bridge.name,
      bridge.dynasty,
      // ... 其他字段
    );
  }
});
insertMany(bridges);
```

## 常见问题

### Q: 数据库文件损坏怎么办？
A: 删除 `bridges.db`，运行 `node init-db.js` 重新初始化

### Q: 端口被占用？
A: 修改 `server/index.js` 中的 `PORT` 值，并同步更新 `vite.config.js` 中的代理目标

### Q: 如何查看数据库内容？
A: 使用 SQLite 工具（如 DB Browser for SQLite）打开 `bridges.db`

### Q: 前端无法连接后端？
A: 检查后端服务是否启动，确认端口 3000 未被占用

## 技术总结

这个后端系统的设计理念是**简单、高效、可靠**：

- **简单**: 使用 Express + SQLite，无需复杂的配置
- **高效**: better-sqlite3 提供高性能数据库操作
- **可靠**: 统一错误处理，事务保证数据一致性

整个系统遵循前后端分离架构，通过 Vite 代理解决开发环境的跨域问题，生产环境则由 Electron 桌面应用统一打包提供。
