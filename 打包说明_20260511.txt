# 桥梁信息系统 - 打包说明

## 当前状态

打包过程需要较长时间（约 3-5 分钟），请耐心等待。

## 已完成的配置修复

1. **ES Module 兼容性修复**：
   - 将 `src/electron.js` 复制为 `src/electron.cjs`
   - 更新 `package.json` main 字段指向 `src/electron.cjs`
   - `.cjs` 扩展名强制使用 CommonJS 格式，绕过 ES Module 限制

2. **electron-builder 配置修复**：
   - 移除了不兼容的 `win.sign` 属性
   - 移除了 `electronDist` 配置，使用默认 Electron 分发

## 手动打包步骤

如果自动打包失败，请手动执行以下命令：

```bash
cd d:\前端\my-project

# 1. 构建前端
npm run build

# 2. 打包 Electron（使用 npm 脚本）
npm run electron:build

# 或者使用 electron-builder 直接打包
npx electron-builder --win --dir
```

## 打包输出位置

打包成功后，可执行文件位于：
```
d:\前端\my-project\release\win-unpacked\桥梁信息系统.exe
```

## 常见问题

### 问题：打包时间过长
- 首次打包需要下载 Electron（约 150MB）
- 可以配置 `electronDist` 使用本地已有的 Electron

### 问题：文件被占用
- 确保 `桥梁信息系统.exe` 没有正在运行
- 关闭所有 Electron 相关进程后重试

### 问题：ES Module 错误
- 确保使用 `electron.cjs` 作为入口文件
- 不要修改项目根目录的 `package.json` 中的 `"type": "module"`
